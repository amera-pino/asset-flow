# AssetFlow 実装計画

## 【最優先共通設計】共通画面レイアウト・ルール（デザインシステム）
今後作成・修正するすべてのメイン画面（「備品一覧」「マイ貸出状況」および今後追加されるすべての画面）は、デザインの一貫性を保ち、無駄な余白によるスクロールを排除するため、以下の【共通3段レイアウト構造】を前提として実装すること。

### 1. 最上段：共通ヘッダーエリア（ロゴ・タイトル・サマリー・ナビを凝縮）
- **左側（サービスロゴとタイトル）**:
  - 1行目に、緑色テキストのサービスロゴ `AssetFlow` を必ず表示すること（デグレによる消失厳禁）。
  - そのすぐ下に、大きな「画面タイトル」（例：備品一覧 / マイ貸出状況）を配置すること。
- **中央（インライン・サマリーカード）**:
  - 画面タイトルの「すぐ右隣（同じ行の並び）」に、3つのサマリーカードをコンパクトなサイズ（ミニマルなバッジ風）で横並びにドッキングさせ、縦の余白を極小化すること。
  - マイ貸出状況画面のサマリー文言は **「ログインユーザー」「承認待ち」「貸出中」** とし、「〜の備品」などの冗長なテキストは省くこと。
- **右側（ナビゲーションボタン）**:
  - 対向画面への遷移ボタンを1つだけ配置すること。**自画面への重複ボタン（デグレ）は完全に排除すること。**
    - 備品一覧画面の右上 ➔ 「マイ貸出状況」ボタンのみ
    - マイ貸出状況画面の右上 ➔ 「備品一覧」ボタンのみ

### 2. 中段：共通フィルターエリア（境界線とコンパクトコントロール）
- サマリーカードの並ぶヘッダー行の直下には、共通で薄いグレーの境界線（`border-b border-slate-200`）を引くこと。
- その下に、余白を詰めながらプルダウンや検索ボックスなどのコントロールを横並び（`flex items-center gap-4`等）で配置すること。
- **コントロールの共通規格（厳守）**:
  - **プルダウン（状態・カテゴリ共通）**: 横幅を `w-full` などの可変幅にせず、双方の画面で完全に一致する【固定幅 `w-44`（176px）】で統一・リサイズすること。
  - **部分一致検索テキストボックス**: どの画面でも横幅を完全に一致させること（【固定幅 `w-64`（256px）】の適用）。
  - **クリアボタン**: 検索テキストボックスの右隣に必ず共通デザインで配置すること。ボタンが押された際は、「その画面を開いた時の初期状態（すべてのフィルター・テキストボックスがデフォルト値）」に完全に復元・リセットする共通ロジックを実装すること。

### 3. 下段：メインエリア（高密度な一覧表）
- メインデータはすべてカード型を廃止し、トップ画面と統一されたデザインの「表（テーブル）形式（`<table>`）」で実装すること。
- 操作列のボタン（「キャンセル」「返却」など）は、文字を絶対に縦に改行させず、1行で美しく収めること（`whitespace-nowrap` の徹底）。

### 4. 最下段：共通フッターエリア（ページネーション規格）
- メインエリア（テーブル）のすぐ直下には、ページネーションコントロールを含むフッターエリアを配置すること。
- **配置・デザインの完全同期（厳守）**:
  - テーブルの横幅の枠内（コンテナ内）に完全に収め、右端のラインが垂直にピシッと揃うように配置すること。
  - **左側**: 「1-10 / 25件」といった現在の表示位置と全件数を示すテキストを配置。
  - **右側**: 「前へ」「次へ」のページ移動ボタン（または矢印アイコンボタン）を配置。
  - ボタンやテキストのフォントサイズ、余白、枠線、背景色、ホバー効果、無効（Disabled）時のスタイルにいたるまで、**先行して実装済みの「備品一覧」のデザインコンポーネントと完全に一致させること。**

ユーザー体験とデータ整合性を優先し、`pending`（承認待ち）状態の申請も「在庫予約」として扱い、二重申請を防ぐ。

**【実装の詳細指示】**
1. **サービス層の新規作成** (`backend/app/services/asset_request_service.py`)
   - `with_for_update()` を使用して、対象の `Asset` 行をロックし、同時実行を防ぐ。
   - 在庫チェック式: `(現在在庫) - (その備品に対するpending状態の申請数量合計) >= (今回の申請数量)`
   - 在庫不足時のエラーメッセージ: 「現在、他の方が申請中のため在庫が不足しています」
2. **APIエンドポイントの実装** (`backend/app/api/routes/requests.py`)
   - `POST /api/requests` を作成。
3. **ルーターの登録** (`backend/app/main.py`)
