## 2. バックエンド基礎実装 [済]
- `Asset`, `AssetRequest` モデルの定義。
- Pydanticによるバリデーション（在庫1以上ルール）の実装。
- 共通レスポンス形式（success/data/error）の導入。
- 備品一覧取得API (`GET /api/assets`) の実装と動作確認完了。

## 3. バックエンドのビジネスロジック実装（申請処理と排他制御） [済]
**【確定した仕様：案1（在庫予約方式）】**
ユーザー体験とデータ整合性を優先し、`pending`（承認待ち）状態の申請も「在庫予約」として扱い、二重申請を防ぐ。

**【実装の詳細指示】**
1. **サービス層の新規作成** (`backend/app/services/asset_request_service.py`)
   - `with_for_update()` を使用して、対象の `Asset` 行をロックし、同時実行を防ぐ。
   - 在庫チェック式: `(現在在庫) - (その備品に対するpending状態の申請数量合計) >= (今回の申請数量)`
   - 在庫不足時のエラーメッセージ: 「現在、他の方が申請中のため在庫が不足しています」
2. **APIエンドポイントの実装** (`backend/app/api/routes/requests.py`)
   - `POST /api/requests` を作成。
3. **ルーターの登録** (`backend/app/main.py`)
   - requests ルーターを FastAPI アプリに追加。

## 4. 初期テストデータ投入およびバックエンドのテスト [済]
- **ファイル作成**: `backend/app/seeds.py` を作成し、SQLAlchemy経由でデータを投入するスクリプトを実装。
- **投入データ案**: 実務リアリティのある備品を5〜10件程度（在庫数にバリエーションを持たせる）。
  - 例: MacBook Pro 14" (在庫3), Dell 27" 4K Monitor (在庫5), HHKB Studio (在庫2), Magic Mouse (在庫10), Herman Miller Aeron (在庫2) 等。
- **実行**: `docker compose exec` 等を利用してコンテナ内でスクリプトを実行し、DBに反映させる。

### 【開発プロセス・ルール】
- **段階的一括実行**:
  1. フロントエンドのUI作成（画面・ナビゲーション・モックデータ）
  2. バックエンドのAPI実装（データ取得・ステータス更新）
  3. フロントとバックのAPI連携
  上記の3ステップについて、各ステップ内は承認不要でコードの生成から適用（Apply）まで最後まで一気に完遂すること。ただし、次のステップへ移る前に、必ずそのステップの「コード変更の全容」をチャットで提示し、人間側の確認を挟むこと。
- **実装内容の報告**:
  各ステップの完了時、修正・新規作成したファイル名と、具体的な変更内容（コンポーネント名、関数名、エンドポイント等）を箇条書きでチャットに明確に報告すること。（※注意：Markdownファイルのステータス変更は人間側で行うため、AIはファイルを直接書き換えないこと）
- **エラー発生時の挙動**:
  コード生成やコンパイル、テスト中にエラーが発生した場合は、隠蔽して無理に修正を続けず、その時点で即座にエラー内容をチャットに報告し、人間の指示を仰ぐこと。

**【確定した仕様：画面の役割分担と暫定認証対応】**
一覧画面のUIをシンプルに保つため、メニューに「マイ貸出状況」画面を新しく作成し、そこで自身の貸出状況の確認と返却処理を完結させる。また、認証機能実装前のため、一連の処理は「user_id = 1」の固定ユーザーとして処理を行う。

**【実装の詳細指示】**
1. **フロントエンド：マイ貸出状況画面の新規作成** (`frontend/src/pages/MyRequestsPage.tsx` 等)
   - ヘッダーのナビゲーションに「マイ貸出状況」のリンクを追加。
   - ログインユーザー（user_id=1固定）が「現在貸出中」の備品をカードまたはリスト形式で表示（まずはコンポーネント内にダミーのモックデータ配列を持たせること）。
   - 貸出中のアイテムが0件の場合は、「現在、あなたが借りている備品はありません。」というプレースホルダーUIを表示。
   - 各アイテムの横に「返却する」ボタンを配置。
2. **バックエンド：返却関連APIの追加** (`backend/app/api/routes/requests.py`)
   - `GET /api/requests/me/active`: user_id=1 かつ status="貸出中" のレコードを全件取得するエンドポイントの実装。
   - `POST /api/requests/{request_id}/return`: 対象の申請履歴の status を "返却済み" に更新し、返却日時（returned_at）を記録するエンドポイントの実装。
3. **フロント・バックエンドのAPI連携**
   - フロントエンドから上記APIを呼び出し、返却成功時にトースト通知を表示させ、画面の貸出リストをリアルタイムに再検証（リロード）する処理の実装。
