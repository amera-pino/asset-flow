# Backend Design Notes

このファイルは、`backend/README.md` とは別に、バックエンドの設計意図やドメインの分け方を残すためのメモです。

## このメモに書くこと

- ルーターをどの単位で分けているか
- 画面とAPIの対応関係
- その設計にした理由
- 学習中に気づいた実装上の考え方

## 今のルーター構成の考え方

- `assets` は備品まわりのAPIをまとめる
- `requests` は申請・貸出まわりのAPIをまとめる
- `マイ貸出状況` は `requests` の中の機能として扱う

画面単位で `include_router` を増やすのではなく、**業務ドメイン単位でAPIを整理する**方針になっている。

## 画面とAPIの対応

- 備品一覧画面 -> `assets.router`
- 申請画面 -> `requests.router`
- マイ貸出状況画面 -> `requests.router`

## README との役割分担

- `README.md`
  - 起動方法
  - 環境構築
  - 毎日の作業手順
  - トラブルシューティング
- `design-notes.md`
  - 設計の意図
  - ドメインの分け方
  - 画面とAPIの対応

## メモ

- 初見で見やすいことと、責務をきれいに分けることは、必ずしも同じではない。
- このリポジトリでは、実装はドメイン単位で整理し、説明は別ファイルで補うのがよさそう。
